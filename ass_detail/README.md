# Plum AI Engineer Assignment — Health Insurance Claims Processing System

## Overview

A multi-agent pipeline that processes health insurance OPD claims: document verification →
extraction → policy evaluation (with canonical mapping) → fraud detection → decision, with a
full explainability trace at every step. See [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md) for
the design and [`docs/COMPONENT_CONTRACTS.md`](docs/COMPONENT_CONTRACTS.md) for every agent's
input/output contract.

## Setup

Requires Python 3.10+.

```bash
python -m venv venv
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

pip install -r requirements.txt
```

Copy `.env.example` to `.env` and add a free-tier Gemini API key (get one at
https://aistudio.google.com/app/apikey):

```
GEMINI_API_KEY=your-api-key-here
```

The Gemini key is **only** needed for the live document upload path (image classification and
extraction). All 12 scenarios in `test_cases.json` and the test suite run without it, via the
injection-mode test harness (see [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md), §8).

## Running things

All commands below assume the venv is activated. On Windows, if activation didn't put `python` on
your `PATH` correctly, use `venv\Scripts\python` in place of `python`.

**Unit + integration tests** (35 tests, no API calls):

```bash
pytest tests/ -q
```

**Evaluation suite** — runs all 12 `test_cases.json` scenarios through the orchestrator in
injection mode (zero API calls) and regenerates [`docs/EVAL_REPORT.md`](docs/EVAL_REPORT.md):

```bash
python -m eval.run_eval
```

**Streamlit UI**:

```bash
streamlit run app/ui/streamlit_app.py
```

This opens three pages:
- **Home** — overview and navigation.
- **Submit Claim** — fill out a claim form and either upload document images (requires
  `GEMINI_API_KEY`) or describe documents directly via the "skip Gemini" fields, then view the
  decision and full explainability trace.
- **Eval Suite** — run the same 12-case evaluation suite from the UI and inspect each case's
  checks, decision, and trace.

**Mock documents** — regenerate the demo document images used with the Submit Claim page:

```bash
python mock_documents/generate_mocks.py
```

This writes JPEGs to `mock_documents/output/` covering a clean approval pair, a patient-mismatch
pair, a wrong-document-type prescription, a blurry/unreadable pharmacy bill, and an alternative-
medicine pair (for the `simulate_component_failure` degraded-pipeline demo).
